CopyPaste — Vocabulary Reference

DATA HIERARCHY
--------------
Payload / payload_v2
  The top-level data object loaded from the DB. Defined in src/main/payload_v2.js.
  Contains metadata (name, mode, saved_at) plus a chunks array and activeChunkIndex.

Chunk
  A named grouping of items within a payload (e.g. "Alpha Flight", "Bravo Squadron").
  Each chunk has a chunk_title and an activeItemIndex tracking which item was last selected.

Item
  A single row within a chunk — represents one production line / SLI / PLI.
  Has an item_title and a contents array.

Contents / Content slot
  The individual fields within an item. Each slot has:
    c_id       — numeric key (0–9 normal layer, 10–19 shift layer)
    c_title    — display label
    c_contents — the value that gets copied to clipboard

LAYERS
------
Normal layer   — c_id 0–9, triggered by Ctrl+1 through Ctrl+0
Shift layer    — c_id 10–19, triggered by Ctrl+Shift+1 through Ctrl+Shift+0

WINDOWS / COMPONENTS
--------------------
Overlay        — src/renderer/src/App.svelte
  The narrow always-on-top panel on the left edge of the screen.
  Handles item navigation, chunk switching, and clipboard copy.

Chunk bar      — the "← [chunk_title] →" strip on the overlay.
  Visible only when the payload has more than one chunk.

Menu / Launch window — src/renderer/src/Menu.svelte
  The main setup window. Contains pages: home, setup (programmatic),
  1x1, payload viewer, quick links, and manage-db.

SQL Table viewer — src/renderer/src/PayloadTable.svelte
  A separate window for inspecting the raw DB contents as a table.
  Supports per-chunk tabs, column sort, and checkbox filtering.

Manage DB page — page within Menu.svelte (page === 'manage-db').
  Save/load/delete .db snapshot files from data/DBs/.

IPC / STATE TERMS
-----------------
setup-complete  — IPC event sent from the menu when a payload is ready.
                  Triggers savePayload → buildPayloadV2 → sli-data to overlay.

sli-data        — IPC channel that pushes the active payload_v2 object to the overlay.

global-nav      — IPC channel for arrow-key navigation originating outside the overlay
                  (Ctrl+Shift+Up/Down/Left/Right). Carries direction: up | down | left | right.

chunkState      — Per-chunk { filters, sortCol, sortDir } map in PayloadTable.svelte.
                  Keyed by chunk_title so sort/filter persist across tab switches.

panelState      — { val → boolean } map inside the filter dropdown.
                  Single source of truth for checkbox state; drives chunkState.filters.

DB TERMS
--------
data/DBs/       — Directory of saved .db snapshot files (better-sqlite3 format).
dummy-2-chunk   — Seeded test DB with two chunks (Alpha Flight, Bravo Squadron)
                  for verifying chunk navigation. Auto-created on first app launch.
